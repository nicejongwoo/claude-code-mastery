# CLAUDE.md

이 파일은 이 저장소의 코드를 다룰 때 Claude Code (claude.ai/code)에게 지침을 제공합니다.

## 프로젝트 개요

이 프로젝트는 클로드 코드(Claude Code)를 활용한 효율적인 개발 워크플로우를 제공하는 가이드 프로젝트입니다. 세 개의 전문화된 서브 에이전트와 명확한 역할 분리를 통해 UI 개발, 기능 구현, 최종 검수를 체계적으로 진행합니다.

## 프로젝트 구조

- **`.claude/agents/`** – 서브 에이전트 설정 파일
  - `ui-component-builder.md` – UI 컴포넌트 개발 담당
  - `feature-implementer.md` – 기능 구현 담당
  - `final-review-coordinator.md` – 최종 검수 담당

## 주요 규칙

- 파일 및 폴더 구조를 명확하게 유지합니다
- 재사용 가능한 컴포넌트를 중심으로 설계합니다
- 중앙 집중식 설정(상수, 테마 등)을 사용하여 일관성을 보장합니다
- TypeScript를 사용하여 타입 안정성을 확보합니다
- 답변은 한글로 작성하고, 입문자가 이해하기 쉬운 형태로 제공합니다
- 비유적인 표현을 사용하지 않고 직접적이고 명확하게 설명합니다

## 서브 에이전트를 통한 개발 워크플로우

이 프로젝트는 세 개의 전문화된 서브 에이전트를 사용하여 효율적으로 개발합니다.

### 1. UI 컴포넌트 빌더 (ui-component-builder)

**언제 사용합니다:**
- UI 컴포넌트를 새로 만들어야 할 때
- 새로운 화면의 레이아웃을 구성할 때
- 기존 UI를 수정해야 할 때

**에이전트의 책임:**
- 컴포넌트는 UI 표현만 담당합니다
- `constants/theme.ts`의 색상과 글꼴을 사용합니다
- TypeScript를 사용하여 type-safe 코드를 작성합니다
- 라이트/다크 테마 모두를 지원합니다

**사용 예시:**
```
사용자: "사용자 정보를 표시하는 프로필 카드 컴포넌트를 만들어줘"
→ ui-component-builder가 프로필 카드 UI를 생성합니다
```

### 2. 기능 구현자 (feature-implementer)

**언제 사용합니다:**
- UI 화면이 완성되었고 실제 기능을 추가해야 할 때
- 상태 관리 로직이 필요할 때
- API 호출 또는 데이터 처리가 필요할 때
- 사용자 인터랙션을 처리해야 할 때

**에이전트의 책임:**
- UI 에이전트가 만든 컴포넌트 구조를 변경하지 않습니다
- Props로 데이터를 전달하고 콜백으로 이벤트를 처리합니다
- 상태 관리, API 통신, 에러 처리를 담당합니다
- 로딩 상태와 에러 메시지를 UI에 반영합니다

**사용 예시:**
```
사용자: "프로필 카드 UI는 완성되었는데, 실제 사용자 데이터를 서버에서 불러와야 해"
→ feature-implementer가 데이터 페칭 로직을 구현합니다
```

### 3. 최종 검수 담당자 (final-review-coordinator)

**언제 사용합니다:**
- UI 개발과 기능 구현이 모두 완료되었을 때
- 배포 전에 전체를 검증해야 할 때
- UI와 기능이 제대로 통합되었는지 확인해야 할 때

**에이전트의 책임:**
- UI와 기능의 통합 검증
- 프로젝트 표준 준수 확인
- 코드 품질 및 성능 검토
- 배포 준비 상태 확인

**사용 예시:**
```
사용자: "프로필 카드 UI와 데이터 페칭 기능이 모두 완성되었습니다. 최종 검수해주세요"
→ final-review-coordinator가 통합을 검증하고 배포 준비 상태를 확인합니다
```

## 역할 분리의 장점

- **전문성**: 각 에이전트가 자신의 영역에 집중합니다
- **속도**: 병렬로 작업할 수 있어 개발 속도가 빠릅니다
- **품질**: 각 단계에서 전문가의 검토가 이루어집니다
- **유지보수**: 역할이 명확하므로 코드 수정이 쉽습니다
